== Overview

skill-stats is a GitHub Action that analyzes Agent Skills directories and produces token cost statistics. It follows the Agent Skills progressive disclosure model to measure how much LLM context a skill consumes at each loading tier.

== Module Responsibilities

|===
| Module | Source | Responsibility

| Main
| `src/main.ts`
| GitHub Action entry point. Reads `@actions/core` inputs, orchestrates the pipeline, writes outputs and job summary.

| Discovery
| `src/discovery.ts`
| Finds Agent Skill directories. Single-skill mode (SKILL.md at root) or multi-skill mode (scan subdirectories).

| Parser
| `src/parser.ts`
| Parses SKILL.md using `gray-matter` for frontmatter extraction. Validates metadata against Agent Skills spec. Enumerates and categorizes all files.

| Analyzer
| `src/analyzer.ts`
| Token counting via tiktoken cl100k_base. Computes progressive disclosure tiers (catalog, instructions, resources). Classifies context budget rating.

| Output
| `src/output.ts`
| Generates Shields.io badge JSON, markdown summary tables, and formatted token counts.

| Types
| `src/types.ts`
| Shared TypeScript interfaces and type definitions.
|===

== Deployment

The action is deployed as a bundled `dist/index.js` file committed to the repository. GitHub Actions runners execute it directly using Node.js 20 (`runs.using: node20` in `action.yml`). No Docker, no installation step.

Users reference it in workflows as:

[source,yaml]
----
- uses: provenimpact/proven-actions@v1
  with:
    path: .agents/skills
    badge-output-dir: ./badges
----
