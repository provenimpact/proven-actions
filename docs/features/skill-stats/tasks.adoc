== Phase 1: Core Types and Discovery

=== Task 1.1: Define shared types [P]
File:: `src/types.ts`
Scenarios:: All (shared data model)
Description:: Create the TypeScript interfaces used across all modules: `DiscoveredSkill`, `SkillMetadata`, `ParsedSkill`, `SkillFile`, `ValidationIssue`, `TierBreakdown`, `FileTokenCount`, `ContextRating`, `SkillAnalysis`, `BadgeJson`.

=== Task 1.2: Implement discovery module [P]
File:: `src/discovery.ts`
Scenarios:: @SS-004, @SS-005
Description:: Implement `discoverSkills()` function. Check if SKILL.md exists at root (single-skill mode) or scan subdirectories (multi-skill mode). Return array of `DiscoveredSkill`.

=== Task 1.3: Unit tests for discovery [P]
File:: `src/discovery.test.ts`
Scenarios:: @SS-004, @SS-005
Description:: Test single-skill detection, multi-skill scanning, skipping non-skill directories, and empty directory handling. Use temp directories with fixture SKILL.md files.

== Phase 7: Local CLI

Scenarios:: @SS-017, @SS-020, @SS-021
Description:: Create a standalone CLI script that reuses Discovery, Parser, Analyzer, and Output modules. Parse `process.argv` for positional path argument (default `.opencode/skills`). Print per-skill summary lines and full markdown summary to stdout. Exit 0 on success.

File:: `scripts/generate-badges.ts`
Scenarios:: @SS-018

=== Task 8.2: Remove committed badges/ directory [x]
Scenarios:: @SS-024
Description:: Delete the `badges/` directory from the repository. Use `git rm -r badges/` to remove tracked badge files.

=== Task 8.3: Update workflow to publish badges to gist [x]
File:: `.github/workflows/skill-stats.yml`
Scenarios:: @SS-023
Description:: Replace the "commit badges to main" step with a gist publishing step. Use a shell script that reads all `badges/*.json` files and PATCHes them to the GitHub Gists API. Scenarios:: @SS-024
Description:: Replace all badge URLs from `raw.githubusercontent.com/.../main/badges/<name>-badge.json` to `gist.githubusercontent.com/{user}/{gist_id}/raw/<name>-badge.json`. Update both the dogfood badge table and the usage documentation pattern.
